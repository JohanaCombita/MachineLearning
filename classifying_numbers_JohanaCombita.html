Introduction to ConvNets: Classifying handwritten numbers¶

Let's take a look at a simple example of a convnet. We will use it to classify the MNIST dataset, which is an open dataset containing handwritten numbers.

Let's create a first basic convnet. It's a stack of 'Conv2D' and 'MaxPooling2D' layers. The important thing to note is that a convnet takes as input tensors of size (image_height, image_width, image_channels). To do this we must first find out the size of the images in our dataset.

The network must have the following layers:

  • A convolutional layer (Conv2D) with 32 3x3 filters and relu activation. In this first layer you must indicate the size of the input (input_shape).
  • A second layer of Max Pooling (MaxPooling2D) of 2x2
  • A third convolutional layer with 64 3x3 filters and relu activation
  • A fourth layer of 2x2 Max Pooling (MaxPooling2D)
  • A fifth convolutional layer of 64 3x3 filters and relu activation

You'll know you've done it right when the model.summary() output is:

Con base en las indiaciones dadas para cada capa, se crea la red. La primera capa aprende 32 filtros con una ventana tamaño 3x3, donde cada filtro se desliza (convolution) sobre la imagen de entrada, produciendo 32 mapas de activación. La activación ReLU (Rectified Linear Unit) se utiliza para añadir no linealidad y ayuda a la red a capturar patrones más complejos. La segunda capa realiza un pooling de 2×2, reduciendo la dimensión espacial de los mapas de activación; donde, al reducir la resolución espacial hace que la red se concentre en características más globales y se reducen los parámetros, evitando sobreajuste y bajando el coste computacional. Luego se aplica otra convolución sobre la salida reducida anterior con 64 filtros 3x3 y la función de activación ReLu, para aprender características más complejas. En una cuarta capa se reduce nuevamente la dimensión especial 2x2; y finalmente, una tercera convolución con 64 filtros de 3x3, donde en esta fase la red ya se encuentra extrayendo rasgos de mayor nivel, combinando la información de las capas anteriores.

# Definición del modelo
model = models.Sequential()

#Capa Conv2D con 32 filtros 3x3, activación ReLU y forma de entrada (28, 28, 1)
model.add(layers.Conv2D(32, (3, 3), activation='relu', input_shape=(28, 28, 1)))
# Capa de MaxPooling2D 2x2
model.add(layers.MaxPooling2D((2, 2)))
#Capa Conv2D con 64 filtros 3x3 y activación ReLU
model.add(layers.Conv2D(64, (3, 3), activation='relu'))
#Capa de MaxPooling2D 2x2
model.add(layers.MaxPooling2D((2, 2)))
# Capa Conv2D con 64 filtros 3x3, activación ReLU
model.add(layers.Conv2D(64, (3, 3), activation='relu'))

#salida
model.summary()

You can see above that the output of each Conv2D and MaxPooling2D layer is a 3D tensor of dimensions (height, width, channels). The width and height tend to decrease as we go deeper into the network. The number of channels is controlled by the first argument passed to the Conv2D layers (e.g. 32 or 64).

The next step would be to give our last tensor (of dimensions (3, 3, 64)) as input to a densely connected network. These classifiers process vectors, which are 1D, while our output is a 3D tensor. So first we will have to flatten our 3D output and convert it to 1D and then add a few dense layers:

  • Add a first layer of 64 neurons
  • Add a last layer of 10 neurons (as many as you can sort) and softmax activation
  • You'll know you've done well when the summary looks like this:

Después de la última capa convolucional se obtiene un tensor 3D de dimensiones (3, 3, 64), donde se contienen características de alto nivel de la imagen. Sin embargo, para poder generar una salida que clasifique los números hay que conectarlo a una capa densa que opera sobre vectores 1D, por lo cual se debe convertir el volumen 3D que sale de la última convolución en un vector 1D. La salida (3, 3, 64) tiene un total de 3 * 3 * 64 = 576 valores, los cuales son aplanados con la función Flatten. Seguidamente, es posible conectar la capa densa con 64 neuronas y la función de activación ReLu, para luego añadir la capa densa con las 10 neuronas de salida para cada número, con una función de activación Softmax. Esta capa es el clasificador que reciba las características extraídas y decida a qué clase pertenece la imagen. Esta última capa cuenta con una función de activación diferente debido a que cada neurona de esta capa corresponderá a la probabilidad estimada de que la imagen pertenezca a la clase x (valores de 0 a 9), y esta función es usada para problemas de clasificación múltiple ya que fuerza a que la salida sea un vector de probabilidades que sumen 1.

Seguidamente, se hace un preprocesamiento de los datos, donde primero se hace una reestructuración de los tensores para que tengan la forma (28,28,1) en lugar de (28,28), es decir, se añade un canal para indicar que la imagen es en blanco y negro. Esto se hace debido a que las capas convolucionales esperan un tensor de entrada con la forma (altura, anchura, canales); por lo cual, es necesario expandir la dimensión para reflejar este canal. Luego, se normaliza cada valor a un rango 0-1 dividiendo entre 255, lo que ayuda a la red a entrenar más rápido y con mayor estabilidad numérica. Esto se hace debido a que las imágenes vienen con valores de intensidad de píxel en el rango 0-255.

De esta manera, se obtendría un modelo más robusto que es capaz de interpretar cambios o ruidos en las imágenes reales para hacer una buena predicción.
